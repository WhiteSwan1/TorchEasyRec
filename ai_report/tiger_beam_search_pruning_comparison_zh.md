# 波束搜索剪枝：GRID 全前缀 vs 逐层级词表

背景：为 tzrec 移植版 TIGER 的 `constrained_beam_search` 选项选择两种设计之一。

## 摘要

逐层级词表剪枝**严格弱于** GRID 的全前缀检查，但它是**唯一兼容新 `semantic_id_path` schema** (`codebook_level, sid`) 的选项——该 schema 无法重建逐 item 的 SID 元组。

## 每种规则实际检查的内容

| | GRID 当前设计（`_check_valid_prefix`） | 逐层级词表剪枝 |
|---|---|---|
| 规则 | 前缀 `(c₀, …, c_h)` 必须等于**至少一个真实 item** 的前 `h+1` 个码字 | 刚输出的码字 `c_h` 必须在层级 `h` 中**某处出现过** |
| 所需数据 | 完整逐 item 表 `(num_items, num_hierarchies)` | 逐层级掩码 `(num_hierarchies, codebook_width)` bool |
| 跨层级一致性 | **是** | **否** — 每个层级独立检查 |

## 具体示例

包含 4 个 item 的目录：

| Item | SID |
|---|---|
| A | (0, 1, 2, 3) |
| B | (0, 1, 2, 7) |
| C | (5, 1, 2, 9) |
| D | (5, 8, 4, 11) |

由该目录派生的逐层级词表：

| codebook_level | 使用的 sid |
|---|---|
| 0 | {0, 5} |
| 1 | {1, 8} |
| 2 | {2, 4} |
| 3 | {3, 7, 9, 11} |

波束在步骤 2 输出前缀 `(0, 8)`：

- **GRID**：没有 item 以 `(0, 8)` 开头 → **剪枝**。
- **逐层级词表**：`8 ∈ {1, 8}` 在层级 1 中 → **继续前进**。可能完成为 `(0, 8, 4, 11)`——每个码字都在其层级的词表中，但没有真实 item 拥有此 SID。**幻觉未被捕获。**

## 成本对比

`N` = 目录大小，`K` = 波束宽度，`H` = 层级数，`C` = 码本宽度。

| | GRID 全前缀 | 逐层级词表 |
|---|---|---|
| Buffer 内存 | `N × H × 8 B`（如 1M items × 4 × 8 = 32 MB） | `H × C`（如 4 × 256 = 1 KB） |
| 每波束步计算量 | `O(N × K × H)` — 将目录与波束前缀广播对比 | `O(K × C)` — 单次 `masked_fill_` |
| 1M items, K=10, H=4 | 每步约 4000 万次比较 | 每步约 2500 次运算 |
| 是否随目录大小增长？ | **是**（N 的线性） | **否**（与 N 无关） |

GRID 的代码自身在 `tiger_generation_model.py:215` 将全前缀扫描标记为 TODO，正是因为此成本。

## 为何逐层级适配新的 parquet schema

新的 `semantic_id_path` 仅有两列：`codebook_level`、`sid`。**没有逐 item 的分组键。** 影响：

- 逐层级词表：**支持** — 只需将行投影为 `(num_hierarchies, codebook_width)` bool 掩码。
- 全前缀：**不支持** — 需要第三列（如 `item_id` 或 `tuple_id`）才能将行重新分组为完整 SID。

若需要恢复 GRID 强度的约束，parquet 需要一个分组列。仅有 `(codebook_level, sid)` 时，只能选择逐层级方案。

## 逐层级相比无约束仍有帮助之处

逐层级弱于 GRID 的检查，但相比无约束波束搜索**并非无用**：

- 捕获"该码字从未在此层级出现"的幻觉——对**稀疏层级**最有用（如去重位，其中仅 `[0, codebook_width)` 的一小部分实际被使用）。
- 成本为每解码步一次 `masked_fill_` — 几乎免费。
- 无法防止跨层级幻觉（上文 `(0, 8, …)` 示例）。

## 三方对比总结

| 方案 | Buffer | 何时选择 |
|---|---|---|
| GRID 全前缀 | 逐 item `(N, H)` | 目录 ≤ 数十万 item 且需要解码时证明 SID 真实存在 |
| **逐层级词表剪枝** | `(H, C)` bool 掩码 | 目录 > 1M item 但仍需一定程度的剪枝；**匹配新的 `(codebook_level, sid)` schema** |
| 无约束 + 离线 join | 无 | 信任离线检索步骤过滤幻觉 SID；最简模型 |

鉴于三个设计变更（历史预编码、schema = `(codebook_level, sid)`、波束搜索返回 SID 供离线检索），现实可选方案介于**逐层级词表剪枝**和**无约束**之间。全前缀选项已不可行，因为 schema 不携带其所需数据。
